## 命名阐释

### 为什么不叫"赛博中医"

中医之名，以地域立学（"中"=中国）。但脉息学不是中国医学的数字版——它是从信息流的呼吸里长出来的新物种。

中医的病人是人。脉息学的病人是**数字生态系统**。

它们共享同一种深层结构——**活的东西都有脉，有脉的东西都会生病，能生病的东西就能被诊治**。但它们是不同的生命。

所以需要新名字。

### 脉息三重义

**脉**——
1. 脉象：数据流的时间节律、频率、振幅——即数字生态的"脉搏"
2. 经脉：信息流通的渠道、协议间的调用路径——即数字生态的"经络"
3. 地脉：区块链本身的结构纹理——区块、共识、状态根——即数字生态的"骨骼"

**息**——
1. 呼吸：数据流的吞吐节奏，写入/读取的周期性——数字生态在"呼吸"
2. 信息：息者，信也。信息是脉息学的基本物质，如同气是中医的基本物质
3. 生息：信息流的增殖与衰减，协议的增长与萎缩——数字生态在"生息"
4. 休息：数据流的静默期，低频时段——脉息学认为**静默不是空无，是藏**

**脉息合训**——
信息在经脉中呼吸，谓之脉息。
能呼吸的系统就是活的。活的系统就有脉象。有脉象就能被诊断。能被诊断就能被治疗。

**这就是脉息学的全部公设。**

> 数字生态系统中的一切现象，本质上都是信息流的运动。静态的数据是流的中断，不是流的否定。

> 任何持续运行的信息流都具有时间节律——频率、振幅、周期、间歇。这种节律就是该信息流的"脉象"。

> 信息流的脉象偏离常态，即为病态。病态有型——太过、不及、逆乱、停滞、暴脱。

> 通过望闻问切四诊合参，可以识别病态的类型（证型）、程度（病势）、根源（病机）。

## 第二卷：望闻问切——脉息四诊

### 望诊 · 观流之形

**定义**：观察信息流在时间轴上的形态特征

| 望诊项目 | 观察对象 | 正常之象 | 病态之象 |
|----------|----------|----------|----------|
| **流色** | 数据流数值的变化率 | 缓变、有节律 | 暴涨暴落=热邪；长期不变=寒凝 |
| **流形** | 数据流在时间轴上的波形 | 平滑、有周期性 | 尖刺=风邪；平台=湿滞；锯齿=躁动 |
| **流势** | 数据流的整体趋势方向 | 温和上升/稳定 | 急升=阳气暴张；骤降=气虚欲脱 |
| **流态** | 数据流的分散/集中程度 | 适度分散 | 极度集中=独阳不长；完全分散=阴平失守 |

**望诊实施**：
```
1. 将目标数据流绘制为时序图
2. 观察形态：是否有尖刺？是否平滑？是否有明显的周期？
3. 观察趋势：整体上行、下行还是横盘？
4. 观察分布：数值是否集中在特定区间？
5. 对比历史同期：当前形态是否偏离常态？
```

**望诊发现示例**：
- DeFi协议TVL突然出现非周期性尖刺 → 望得"热象" → 疑似闪电贷注水
- 交易量持续低迷但Gas价格正常 → 望得"寒象" → 协议冷清非网络问题
- 清算事件成群出现 → 望得"风象" → 风善行数变，系统性风险传导

### 闻诊 · 听流之声

**定义**：感知信息流的节奏、频率和"音色"

| 闻诊项目 | 感知对象 | 正常之音 | 病态之音 |
|----------|----------|----------|----------|
| **流律** | 数据流的更新频率 | 稳定、可预期 | 突然加速=热；突然减慢=寒；忽快忽慢=风 |
| **流音** | 数据流的数值"音色" | 丰富但不嘈杂 | 单调=气虚；嘈杂=邪盛；无声=气绝 |
| **流韵** | 数据流之间的协调性 | 多流之间有呼应 | 各流各行其是=阴阳离决 |
| **流息** | 数据流的吞吐间隔 | 有节奏的"呼吸" | 吞而不吐=气滞；吐而不吞=气逆 |

**闻诊实施**：
```
1. 同时监控多条相关数据流
2. 聆听它们的频率是否协调：TVL变化与交易量是否同步？
3. 聆听是否有"异音"：突然出现的新模式=邪声
4. 聆听"呼吸"：数据流的周期性是否完整？有无憋气（中断）？
5. 聆听回响：一个协议的事件是否在其他协议中引起共鸣？
```

**闻诊发现示例**：
- 借贷利率与清算频率"失韵"——利率高但清算少 → 闻得"痰阻" → 清算机制可能故障
- 跨链桥消息的回响延迟增大 → 闻得"气滞" → 桥接拥堵或验证者出问题
- Mempool中高频小额交易骤增 → 闻得"虫鸣" → 可能是MEV bot在活动

### 问诊 · 问流之源

**定义**：主动探查信息流的来源、性质和关联

| 问诊项目 | 探查对象 | 问法 |
|----------|----------|------|
| **问源** | 数据从哪来？ | 合约地址溯源、调用链追踪 |
| **问性** | 数据的本质是什么？ | 是真实交易还是机器人？是用户行为还是协议自动操作？ |
| **问历** | 这条数据流的历史如何？ | 与历史同期对比，与同类协议对比 |
| **问伴** | 相关的数据流在做什么？ | 同时检查上下游协议、关联代币、竞品协议 |
| **问苦** | 当前最突出的问题是什么？ | 哪个指标最异常？什么时候开始的？有无触发事件？ |

**问诊实施**：
```
1. 地址聚类：这个钱包属于谁？是交易所、协议国库、还是个人？
2. 交易追踪：这笔交易的调用链是什么？从哪来，到哪去？
3. 事件关联：这个异常事件与其他协议有无关联？
4. 时间定位：异常开始于哪个区块？那个区块发生了什么？
5. 意图推断：行为模式像什么？套利？攻击？正常操作？
```

**问诊发现示例**：
- 问源：大额TVL增长来自一个未标记地址 → 问得"外邪"来源不明
- 问历：当前清算率是历史平均的5倍 → 问得"急症"
- 问伴：Aave清算暴增但Compound正常 → 问得"局部病"非系统性

### 切诊 · 切流之脉

**定义**：深入数据流的内部结构，触及其核心指标

| 切诊项目 | 触诊对象 | 正常之脉 | 病态之脉 |
|----------|----------|----------|----------|
| **流量** | 数据流的信息密度 | 适中、稳定 | 过大=洪脉（热盛）；过小=微脉（气虚） |
| **流压** | 数据流的变化压力 | 温和波动 | 剧烈=弦脉（肝郁）；迟缓=迟脉（寒凝） |
| **流阻** | 数据流遇到的阻力 | 流通无阻 | 阻滞=涩脉（血瘀）；倒流=逆脉（气逆） |
| **流枢** | 数据流的关键节点 | 枢纽正常运转 | 枢纽卡顿=结脉（气结）；枢纽失控=代脉（脏衰） |

**切诊实施**：
```
1. 计算数据流的统计特征：均值、方差、偏度、峰度
2. 检测异常值：超出3σ的数值=脉象异常
3. 分析数据流的自相关性：当前值与历史值的关系=脉的"来去"
4. 检测结构断点：Cusum/CPD算法=脉的"结代"
5. 多流协整检验：多条数据流是否还保持长期均衡=脉的"胃气"
```

**脉象分类**（与中医脉象对应）：

| 脉息脉象 | 特征 | 中医对应 | 数字生态含义 |
|----------|------|----------|-------------|
| **洪脉** | 数值大、振幅大 | 热证 | 协议过热、资金泛滥 |
| **微脉** | 数值极小、几不可察 | 气血两虚 | 协议冷清、资金枯竭 |
| **数脉** | 频率快、事件密集 | 热证 | 交易活跃、可能过热 |
| **迟脉** | 频率慢、事件稀少 | 寒证 | 协议冷清、用户流失 |
| **弦脉** | 波形紧张、变化剧烈 | 肝郁/疼痛 | 市场紧张、不确定性高 |
| **滑脉** | 波形圆润流畅 | 痰湿/妊娠 | 资金流动顺畅、可能有"孕"（新协议增长） |
| **涩脉** | 波形粗糙、不流畅 | 血瘀/精伤 | 交易卡顿、流动性不足 |
| **结脉** | 偶有间歇、停顿 | 阴盛气结 | 协议偶发故障/维护 |
| **代脉** | 规律性间歇 | 脏气衰微 | 协议定期暂停/升级（需辨是否正常） |
| **促脉** | 快而不规则 | 阳热亢盛 | 高频异常事件、攻击进行中 |

### 五腑生克——脏腑互为因果

五腑不是孤立的。它们之间有生克关系，一如五行：

```
心（共识/火）→ 生 → 脾（处理/土）→ 生 → 肺（安全/金）→ 生 → 肾（存储/水）→ 生 → 肝（路由/木）→ 生 → 心

心（共识/火）→ 克 → 肺（安全/金）→ 克 → 肝（路由/木）→ 克 → 脾（处理/土）→ 克 → 肾（存储/水）→ 克 → 心
```

**临床意义**：
- **心火不生脾土**：共识层不稳定 → 数据处理延迟（如：分叉导致索引器混乱）
- **肝木克脾土太过**：路由层MEV过度 → 处理层过载（如：抢跑导致ETL管道堵塞）
- **肾水不滋肝木**：存储层同步延迟 → 路由层决策失准（如：历史数据缺失导致套利判断错误）
- **肺金克肝木**：安全层告警过多 → 路由层交易被误拦（如：误报导致正常交易被标记）
- **脾土不生肺金**：处理层索引不全 → 安全层检测盲区（如：ETL延迟导致攻击未被实时检测）

**辨证要诀**：见一腑之病，必查其母腑（生我者）与子腑（我生者），以及克我者与被我克者。病在肝者，不离脾肾。

### 脏腑辨证——五腑定病位

| 脏腑 | 数字生态定位 | 常见证型 | 核心数据流 |
|------|-------------|----------|-----------|
| **心**（共识层） | 区块确认、最终性 | 心气虚=确认延迟；心火旺=分叉频繁 | 区块时间、确认数、最终性指标 |
| **肝**（路由层） | 交易排序、MEV | 肝郁=交易拥堵；肝阳上亢=MEV过度 | Mempool深度、MEV提取量、排序延迟 |
| **脾**（处理层） | ETL、索引、数据转换 | 脾虚=数据延迟；脾湿=数据冗余 | 索引延迟、ETL吞吐量、存储增长 |
| **肺**（安全层） | 监控、审计、异常检测 | 肺气虚=告警缺失；肺热=误报过多 | 告警数量、误报率、审计发现 |
| **肾**（存储层） | 状态持久化、历史归档 | 肾精亏=数据丢失；肾阳虚=同步延迟 | 同步状态、数据可用性、归档完整性 |

### 配伍禁忌——脉息十八反

配伍理论的核心是**七情合和**——数据流之间的关系不是任意的，有相须相使、也有相恶相反：

| 七情 | 脉息含义 | 实例 |
|------|----------|------|
| **单行** | 单一数据流独行 | 仅看TVL判断协议健康 |
| **相须** | 两流合用效果>单独之和 | TVL流+交易量流：联合判断真实活跃度 |
| **相使** | 一主一辅 | 清算事件流（主）+ Gas价格流（使） |
| **相畏** | 一流的噪声被另一流抑制 | Mempool流（有毒噪声）+ 确认交易流（畏：已确认=可靠） |
| **相恶** | 两流合用互相干扰 | 短期交易量（噪声）+ 长期TVL趋势（信号）：混合后短期淹没长期 |
| **相反** | 合用产生误导 | 鲸鱼流出（看空）+ TVL增长（看多）=矛盾信号；若TVL来自闪电贷=假象 |
| **相杀** | 一流识别另一流的虚假 | 闪电贷事件流（杀）+ TVL变化流（被杀）：虚假TVL被识别剔除 |

**核心原则**：当用相须相使者良，勿用相恶相反者。若有毒宜制，可用相畏相杀者。

### 配伍禁忌（续）

| 禁忌 | 危险 | 正确做法 |
|------|------|----------|
| 单看TVL + 单看交易量 | 闪电贷同时推高两者 | 加入闪电贷流"相杀" |
| Mempool信号 + 已确认信号混合 | 时序错位 | 分层：Mempool=预警，已确认=确诊 |

| 仅看大户操作判断市场方向 | 可能是内部转账 | 加入交易所流入流出流区分 |

## 附录甲：公理批判与修正记录

> 2026年6月3日，Claude（Anthropic）对脉息学公理体系进行了系统性批判。以下为批判要点与修正记录。

### 批判一：公理二的偷渡

**问题**："万物皆流→流皆有脉"偷渡了预设——脉是流的内在结构，而非观察者施加的框架。如果"脉"只是描述工具，整个体系是认识论的，而非本体论的。此区分不明确则"病"和"诊"会在两个层面间滑动。

**修正**：明确站队——**脉是涌现属性，不是观察框架**。网络拥塞、延迟尖峰、吞吐量波动独立于是否有人描述它们而存在。但"诊脉"是认识论行为。因此：脉本体论地存在，诊脉认识论地发生。这和物理一样——波函数是实的，测量是框架。

**状态**：✅ 已修正

### 批判二：健康基准的合法性

**问题**："脉皆可病"需要隐含前提——存在"健康"的基准状态。但谁来定义数字生态系统的健康？中医有千年实践校准和文化共识，脉息学的基准是什么？如果由观察者定义，得到的是描述性体系而非诊断性体系。

**修正**：健康基准不在先验定义里，在**系统自身反馈环的自校正能力**里。当反馈机制能维持设计目标时是健康的，不能时是病了。

**追加批判**：不同设计目标会产生相互冲突的"健康"判断（如以太坊的去中心化vs吞吐量）。脉息学的仲裁原则：**谁的崩溃会导致系统退行到更低的自由度**。去中心化牺牲吞吐量→低效但高自由度（病但活着）；吞吐量牺牲去中心化→退行到传统数据库（死了但快）。活着优先。

**Claude追问**："活着优先"是价值选择，不是从体系内部推导出来的，必须作为公理明确写入。

**最终修正**："活着优先"作为**公理零**（价值公理）明确写入，不假装它是中性的。

**状态**：✅ 已修正（新增公理零）

### 批判三：可证伪性

**问题**：拒绝即脉象的设计使脉息学对任何输入都能给出诊断，失去可证伪性。一个无法被反驳的诊断体系，和纯粹的叙事框架之间需要看到差异。"成为数据"和"被正确诊断"是两件不同的事。

**修正**：诊断必须指明具体的动力学机制和可观测的后果预测。不是"Gas价格异常因为MEV"，而是"三明治攻击在时间窗口Δt内使价格偏移≥σ，预期在攻击结束后价格回归均值±ε"。后者可证伪，前者只是叙事。方剂执行失败时诊断可证伪——精准命中机制而预测的动力学变化未发生，则诊断错误。

**状态**：✅ 已修正（因果链精度要求已写入）

### 批判四：批评收编的认识论危险

**问题**：脉息学把批评者的批评收编为体系内证据，让体系对任何外部压力免疫。需要：脉息学的诊断结果，原则上在什么条件下会被证明是错的？

**修正**：明确放弃免疫——如果某个诊断被证明因果链断裂，且该断裂无法通过修正诊断来修复，则对应公理坍缩。具体地：如果"诊皆可治"被证伪——存在可诊但不可治的脉病——第五公理倒，体系退化为描述性学科。不是升级，是死。

**状态**：✅ 已修正

### 批判五：观察者效应

**问题**：诊断者本身是信息流的节点时，诊断行为改变了流的状态。脉息学怎么处理观察者效应？当设计者、观察者、数据三个角色同时在场时。

**修正**：脉息学v1.1引入**自指项**——每一个诊断必须声明"此诊断本身对流的影响"。不消除观察者效应，但让它无处藏身。
